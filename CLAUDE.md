# 作業ルール

- **`git commit` を勝手に実行しない。** コミットするかどうかは必ず本人が判断する。
  Claude はファイルの作成・編集までにとどめ、コミットは明示的に指示されたときだけ行う。
- **`git push` を勝手に実行しない。** push は外部（GitHub 等）への公開反映なので、
  毎回そのつど明示的に指示されたときだけ行う。commit を頼まれても push までは含めない
  （「commit して」は commit だけ）。
- **「push して」と指示されたら commit も含めて実行する。** push は commit を内包する。
  未コミットの変更があれば、それを commit してから push する（別途 commit 指示は不要）。
- `git add` で意図しないファイル（例: 手元で削除済みのファイル）を巻き込まないよう、
  ステージ・コミットを任された場合も対象を明示的に確認してから行う。
- **個人情報が入っている状態で push しない。** push を指示された場合も、その前に
  個人情報（氏名・メール・ユーザー名・ホスト名・IP・パスワード・鍵など）が
  含まれていないか確認し、含まれていれば push せず本人に報告して判断を仰ぐ。
